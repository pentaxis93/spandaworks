## February 2023: CS50

**Purpose:** Show the choice and the experience

### The Choice

- February 2023, age 53
- Harvard's CS50—intro course, from scratch
- "Hello, World" in C
- Could have kept tinkering, could have done bootcamp, could have jumped to React
- Chose to go back to college course format
- Going back to where he created first foundation

### The Philosophy

- David Malan explaining C before Python
- Higher-level languages hide "dirty memory storage, cleanup, low-level management"
- "We have to learn what's happening first"
- Understand what's being hidden, because "every once in a while, it makes a difference"
- "I appreciated that very much. So I loved that, loved the whole philosophy of the course, actually."

### The Intensity

- February 1-24, 2023: 24 days
- 31 problem sets and labs
- Daily work
- C, Python, SQL, HTML/CSS, Flask
- "Hello, World" to stock trading web app
- Still quite sick, but well enough to focus

### What It Actually Did

- Started the habit: coding daily
- Started the pattern: always having a project
- Reconnected with who he was before illness
- "A kind of healing for me"
- "Took me back to my college days and reconnected me with who I was then"
- Creating new foundation by returning to format of first foundation

**Key point:** Not just learning content—recovering identity, establishing disciplines

## What CS50 Gave (Not What You'd Expect)

**Purpose:** Shift from "learned Python/C" to "learned patterns/habits/disciplines"

- Not using Python anymore. Doesn't matter.
- After CS50: BloopTwing, BibleTLDR (experimental)
- Then Fifth Avenue Gallery (Next.js)—learned TypeScript "not that hard"
- Then Dart—"a lot of new concepts... took a while, but not that hard"
- Then Rust (current)—using AI to learn
- Method: generate useful code, have agent explain line by line

**The pattern visible:**
- Content changes: Python → TypeScript → Dart → Rust
- Foundation remains: patterns, ways of thinking, habits
- Each new language easier because foundation solid

**Quote from walk:** "It's not about the specific content... it was never the specifics that really mattered to me, but the patterns, the ways of thinking, the habits."

**What actually transferred:**
1. Daily coding habit
2. Always having a project
3. Foundation-first thinking
4. Disciplines that now make AI-assisted learning more powerful

## The Counterfactual

**Purpose:** Make the choice explicit, show what was avoided

### If I'd Just Kept Tinkering

- Could have kept building from May 2022

- But: no daily habit, no systematic approach, no foundation

### If I'd Chosen a Bootcamp

- Framework-first, job-focused
- Would have learned different things—maybe gotten job faster
- But: frameworks change, jobs change
- Without patterns/habits/disciplines, each shift starts from scratch

**What CS50 choice avoided:** Being dependent on specific content, locked to specific stack

## The Teaching

**Purpose:** Make the lesson explicit and transferable

### What This Story Is Really About

Not: "Take CS50 at any age"
Not: "Learn foundations before frameworks"

**Actually:**

1. **When you need to rebuild (career, identity, capability), go back to where you built before**
   - CS50 wasn't just learning—it was healing
   - College course format reconnected with who he was when he first built
   - Creating new foundation by returning to site of first foundation

2. **Content doesn't matter—patterns, habits, disciplines do**
   - Not using Python anymore
   - Languages change, frameworks change, stacks change
   - What transfers: ways of thinking, daily habits, systematic approach
   - These make each new learning easier

3. **Foundation makes new modes of learning possible**
   - CS50 habits now enable AI-assisted learning
   - Can learn Rust by generating code and understanding it
   - Foundation provides context, questions, judgment
   - AI provides speed, examples, explanations

4. **Age isn't the barrier—calcification is**
   - 53 → 56, participating in AI emergence
   - "Incredible opportunity"
   - Stay flexible, fight the hardening
   - Keep learning, keep building

### Who This Is For

- Someone who coded once, decades ago
- Life happened (illness, family, career shift, whatever)
- Thinking about returning
- Wondering if too old, too late, too far behind

**The message:** You can rebuild. Go back to where you built before. The patterns and habits will transfer. And they'll serve you in ways you can't predict yet.
