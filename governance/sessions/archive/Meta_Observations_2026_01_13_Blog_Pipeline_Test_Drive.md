## Process Patterns

### Deliberation → Decision → Documentation
Clean three-phase flow:

1. **Deliberation:** Present options with trade-offs
2. **Decision:** Human chooses WHAT (user provided "1. save and delete, 2. option C, 3. A")
3. **Documentation:** System implements HOW and records reasoning

No contamination. User didn't have to articulate WHY, system didn't guess WHAT.

### Test-Drive Methodology
"Test-drive the pipeline" was brilliant framing. Instead of theoretical review:
- Real article
- Real workflow execution
- Real friction discovered
- Real fixes applied

Friction became curriculum. The `editors-table` skill loading failure wasn't a bug—it was a teaching moment about OpenCode skill requirements.
